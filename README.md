# Team Sub-3 ｜ AI Agent活用の実践ポートフォリオ

マラソン「サブ3」達成を目標に、役割ごとに最適なAIを組み合わせて自作したトレーニング管理アプリケーションの実践ポートフォリオです。

---

## 📋 プロジェクト概要

本プロジェクトは、個人開発を**企業のAI Agent導入支援に近い「型」**で運用する実証プロジェクトです。
フルマラソン・サブ3（3時間切り）を目指すトレーニング管理アプリを題材に、**課題定義・Agent設計・データ設計・実装・運用改善**を一気通貫で検証しています。

単なるデモ用コードではなく、**Garmin（ウェアラブル）・食事・睡眠のデータを毎日記録し、AIの判定と提案のもとで実際に走り続けている「生きた運用」**を行っています。

---

## 👥 推進体制（AIエージェントチーム）

人間を Product Owner（PO）に置き、役割ごとに特化した複数のAIエージェントをアサインした「協調型」の体制をとっています。

| 役割 (Role) | AI / ツール | 担当業務内容 |
| :--- | :--- | :--- |
| **Product Owner** | 人間（本人） | 目的・KPI設計、優先順位決定、最終意思決定 |
| **PMO** | Claude | タスク分解、進捗管理、論点の整理 |
| **Engineer** | Codex | 実装案の策定、コードの修正案作成 |
| **Developer** | Antigravity | コードの自律開発、バグ修正、レイアウト調整 |
| **Analyst** | Gemini | トレーニング指標設計、データの異常値確認 |
| **Reviewer** | Claude | 実装コードの品質検証、セキュリティチェック |

---

## 🛠 技術構成

- **データソース**: Garmin（心拍・睡眠・負荷データ）、Omron（体組成データ）
- **データベース**: Supabase
- **バックエンド/連携**: Google Apps Script (GAS)
- **エディタ/ドキュメント**: Obsidian
- **デプロイ/開発**: GitHub, clasp
- **AI/LLM**: Gemini API, Claude, Codex, Antigravity
